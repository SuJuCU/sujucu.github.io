JavaScript在发送Ajax请求时，URL的域名地址是使用相对地址还是绝对地址
发表于2023-10-26|javaweb作业
相对地址：使用相对地址时，请求的 URL 是相对于当前页面的地址。例如，如果当前页面的 URL 是 http://example.com/page1.html，而你发送的 Ajax 请求的 URL 是 /api/data，那么实际请求的地址会是 http://example.com/api/data。相对地址通常用于与当前页面在同一域下的请求。 绝对地址：使用绝对地址时，请求的 URL 包含完整的协议、域名和路径信息。例如，你可以直接使用 http://example.com/api/data 作为 Ajax 请求的 URL。绝对地址通常用于与当前页面不在同一域下的请求，或者需要明确指定请求的目标地址。 选择相对地址还是绝对地址取决于你的具体需求。如果你需要与当前页面在同一域下进行请求，或者需要根据当前页面的地址动态构建请求 URL，那么使用相对地址更为方便。如果你需要明确指定请求的目标地址，或者与当前页面不在同一域下进行请求，那么使用绝对地址更为合适。

如何理解信息隐藏和局部化？
发表于2023-04-20|软件工程作业
信息隐藏：将内部的细节封装起来，提供外部使用的接口，使用时不需要内部的信息。对于其他不需要这些信息的模块来说，信息是不可被访问的。 局部化：模块内部数据只放在一个范围内，减少外部的依赖性。 例子： 校园一卡通的设计方案里，可以把消费记录数组暴露给外部访问，这种方式简洁但是有不必要的耦合：访问校园卡类的其他类将以来校园卡类中的消费记录数组，这意味着如果校园卡类修改内容，其他类也将不得不修改。对查询操作进行抽象和隐藏后，其他类只需要通过方法就可以获得消费记录的条数和指定序号的消费记录，即使校园卡类修改内部数据结构，只需要提供这两个方法，其他类无需修改。

什么是模块化？为什么要模块化？
发表于2023-04-14|软件工程作业
什么是模块化？模块化就是把程序划分成可独立命名且独立访问的模块，每个模块完成一个子功能，把这些模块集成起来构成一个整体，可以完成指定的功能满足用户的需求。 为什么要模块化？如果一个软件仅由一个模块组成，它将很难被人所理解。如果一个问题的复杂度很高，所需要解决的时间也会变多，把复杂的问题分解成小问题时，原来的问题也就容易解决了。 这并不是说如果无限分割软件，最后为了开发软件而需要的工作量可以小得忽略不计，实际上还有另一个因素在起作用。当模块数目增加时，每个模块的规模将减小，开发单个模块所需要的成本减小了。但是模块数目增加，设计模块间接口所需要的工作量也将增加。根据这两个，得出总成本曲线。每个程序都相应地有一个最适当的模块数目M，使得系统的开发成本最小。
